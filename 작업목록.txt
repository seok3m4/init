<E1>

Scope
AI worker/SQS 처리 기반 구축 [backend/worker, SQS]
AI 작업 로그 생성/상태 전이 [ai_process_logs]
AI 작업 상태 PENDING → RUNNING → COMPLETED/FAILED 기록 [ai_process_logs]
AI 출력 안전성 검증 [/ai/guardrails/validate]
가드레일 검증 로그 기록 [ai_guardrail_logs]
실패 사유와 재시도 가능 여부 기록 [ai_process_logs, ai_guardrail_logs]
API 서버와 worker 간 장기 작업 분리 [backend/api, backend/worker]
AI provider/S3/SQS 환경변수 연동 [infra, backend/worker]

Acceptance Criteria
[ ] 장기 AI 작업은 API 서버 요청 스레드에서 직접 오래 실행되지 않는다.
[ ] AI 작업 생성 시 ai_process_logs에 작업 유형, 입력 참조, 초기 상태가 기록된다.
[ ] worker는 작업 시작 전 RUNNING, 성공 시 COMPLETED, 실패 시 FAILED 상태를 기록한다.
[ ] 실패 시 실패 사유와 재시도 가능 여부가 추적 가능하다.
[ ] AI 결과는 POST /ai/guardrails/validate 검증을 통과하기 전 최종 저장되지 않는다.
[ ] 가드레일 결과는 PASS, BLOCKED, REGENERATED 중 하나로 기록된다.
[ ] S3/SQS/AI provider secret은 환경변수로 주입되고 git에 포함되지 않는다.
[ ] A 담당 인프라 설정과 worker 배포/재시작 경계가 맞는다.

Scope

파일 원본 S3 참조 기반 처리 [file_assets, S3]

Acceptance Criteria
[ ] 서류 추출은 S3 파일 참조를 입력으로 사용하고 원본 파일을 DB에 저장하지 않는다.
[ ] 서류 추출 시작/성공/실패 상태가 application_documents와 ai_process_logs에 반영된다.
[ ] STT 처리 결과는 해당 interview_answers.transcript에 저장된다.
[ ] STT 실패 시 ai_process_logs.status=FAILED와 실패 사유가 기록된다.
[ ] 꼬리질문은 이전 질문, 답변 스크립트, 직무/JD/서류 요약 등 필요한 입력을 기반으로 생성된다.
[ ] 모의면접 꼬리질문과 채용면접 꼬리질문은 정책과 표현이 분리된다.
[ ] AI 생성 결과는 가드레일 통과 전 최종 저장되지 않는다.
[ ] 중복 STT/중복 꼬리질문 요청은 상태 충돌 또는 기존 결과 반환 정책으로 처리된다.

Scope

평가 근거 저장 [report_evidences]
채용 리포트와 모의면접 리포트 표현 정책 분리 [report_type]
가드레일 통과 후 최종 저장 [ai_guardrail_logs, evaluation_reports]

Acceptance Criteria
[ ] 평가 컨텍스트는 회사, 공고, 평가 기준, 지원서, 면접 답변, 수동 평가 정보를 기준으로 구성된다.
[ ] 답변 평가 점수는 평가 기준과 연결되고 근거 없는 점수는 저장되지 않는다.
[ ] 평가 근거는 report_evidences에 답변 또는 서류 근거와 함께 저장된다.
[ ] 커뮤니케이션 분석은 보조 지표로만 저장되고 평가 결정 근거로 과대해석되지 않는다.
[ ] 채용 리포트는 RECRUITING_REPORT, 모의면접 리포트는 MOCK_INTERVIEW_REPORT로 구분된다.
[ ] 모의면접 리포트에는 합격/탈락 판단 표현이 포함되지 않는다.
[ ] 리포트 생성 중에는 GENERATING, 성공 시 COMPLETED, 실패 시 FAILED 상태가 반영된다.
[ ] AI 결과는 가드레일 통과 전 evaluation_reports, report_scores, report_evidences에 최종 저장되지 않는다.
[ ] 리포트 생성 실패 시 재시도 가능한 사유와 불가능한 사유가 구분되어 기록된다.

질문/평가 기준 생성 결과 저장 전 검증 [criterion_tags, evaluation_criteria, question_bank]
임베딩 생성 및 중복 방지 [embeddings]
원문 해시 기반 임베딩 중복 생성 방지 [source_text_hash]
AI 생성 결과 가드레일 검증 연동 [ai_guardrail_logs]
C/D 화면에서 사용할 생성 결과 상태 반환 [ai_process_logs]

Acceptance Criteria
[ ] 평가 역량 태그 추천은 JD, 인재상, 평가 정책을 입력으로 사용한다.
[ ] JD 기반 질문 생성 결과는 질문 뱅크에 저장 전 사용자가 검토/확정할 수 있다.
[ ] 질문 세트 구성은 공고, 평가 기준, 질문 유형, 질문 수 조건을 반영한다.
[ ] 모의면접 질문 생성은 채용 질문과 달리 JD/기업 평가 기준 의존 없이 동작한다.
[ ] AI 생성 결과는 가드레일 검증을 통과하기 전 최종 저장되지 않는다.
[ ] 임베딩은 source_text_hash 기준으로 중복 생성되지 않는다.
[ ] 생성 요청은 ai_process_logs로 추적 가능하고 실패 시 사유가 기록된다.
[ ] C 담당 면접 관리 화면과 D 담당 면접 런타임이 생성 결과를 소비할 수 있다.
